Métodos Get e Post

Quando em um formulário, dentro do form, não se determina um método de envio de dados, por padrão, a HTML5 define o método de envio "GET".

O método de envio GET permite que os dados do formulário apareça na URL do site.

Quando se desja enviar dados de um formulário sem que na URL apareça os dados contidos no formulário, usa-se o método "POST". O método POST oculta os dados de um formulário da URL do site, mas não protege estes dados de serem acessados.

Para acessar tais dados basta clicar com o botão direito do mause em cima do formulário aberto, ir para a aba inspecionar abrindo do Devtools. No Devtools clique na opção NETWORK (rede). Procure por "names", "cadastro.php", clique na aba HEADERS e você terá acesso aos dados do formulários.

O método GET é usado quando os dados do formulário não for confidenciais, não ultrapassar mais de 3000 bytes, que não tenha imagens ou arquivos.

Para dados que necessitam ser ocultados utiliza-se o método POST. Este método permite o envio de dados com mais de 3000 bytes, imagens e arquivos.

Para dados confidenciais utiliza-se a criptografia dentro do protocolo https.
