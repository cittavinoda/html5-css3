Select, Datalist e Textarea

A tag select cria uma caixa de seleção para o desenvolvedor montar uma lista de opções para o usuário. No caso em estudo será montada uma lista de Estados do Brasil.

Dentro datag select, utiliza-se a tag option que permite ao desenvolvedor cirar as opções para seleção de estados recebendo um valor atribuído ao nome do Estado em questão.

Os estado são digitados ficam na ordem da digitação. É dever do desenvolvedor colocá-los em ordem alfabética.

Os Estados digitados podem ser organizados por meio de agrupamentos, por exemplo, agrupando-os por regiões. Para isso, as tagas options devem ser envelopadas, oudigitadas dentro da tag "optgroup". A tag "optgroup" recebe o atributo "label" onde se define a região de cada grupo de estado.

Para deixar na caixa de controle um estado selecionado, basta inserir depois do valor dado ao "value" a tag "select".

Elemento Datalist
Definição e Uso

A <datalist> tag especifica uma lista de opções predefinidas para um <input> elemento.

A <datalist> tag é usada para fornecer um recurso de "preenchimento automático" para <input> elementos. Os usuários verão uma lista suspensa de opções predefinidas à medida que inserem dados.

O atributo <datalist> id do elemento deve ser igual ao <input> atributo list do elemento (isso os une).

A tag "datalist" deve conter um "id" com o intuito de interligar esta tag ao elemento "input text" através do atributo "list" inserido na tag "input text" com o valor de identificação da "datalist" - lstprof que significa lista de profissões.

Outro recurso que o elemento "datalist" permit, é que, caso não haja na listagem do programador o registro de uma profissão, o usuário poderá digitar sua profissão na caixa de controle e enviar seus dados normalmente. Após o envio, o elemento "datalist" inseri automaticamente a profissão digitalizada pelo usuário em sua listagem.
